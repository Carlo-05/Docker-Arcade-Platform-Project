## **1. Build Docker Images and push it to your Docker Hub Repo.**
-   Make sure that your are in images directory which we cloned earlier alongside with OIDC directory.
-   Check out and review the Dockerfile in both the current directory and the webpage directory to gain better understanding.
-   Clone the games that we will use in this project:
    
    ```bash
    git clone https://github.com/nebez/floppybird
    git clone https://github.com/wayou/t-rex-runner
    git clone https://github.com/jakesgordon/javascript-tetris
    git clone https://github.com/reruns/mario
    ls -l
    ```
    -   The directory should looked like this:

    <div align="left">
    <img src="https://github.com/Carlo-05/Docker-Arcade-Platform-Project/blob/main/documents/pictures/Execute_001.png?raw=true" alt="Execute"style="width: 30%; height: auto;">
    </div>


-   Copy the Dockerfile into each game directory. The webpage directory already has its own Dockerfile.
   
    ```bash
    cp Dockerfile floppybird/
    cp Dockerfile t-rex-runner/
    cp Dockerfile javascript-tetris/
    cp Dockerfile mario/
    ```

-   Set up Docker Buildx builder prior to executing image builds.
    -   Docker Buildx enable advanced image-building capabilities beyond the standard Docker build workflow.

    ```bash
    docker buildx create --name <desired_name> --use --bootstrap --driver docker-container
    docker buildx ls
    ```

    <div align="left">
    <img src="https://github.com/Carlo-05/Docker-Arcade-Platform-Project/blob/main/documents/pictures/Execute_002.png?raw=true" alt="Execute"style="width: 60%; height: auto;">
    </div>


-   Build the games and webpage images, then push them into your Docker Hub repository.

    ```bash
    docker buildx build -t carlo005/webpage:latest ./webpage/ --push
    docker buildx build -t carlo005/flappy:latest ./floppybird/ --push
    docker buildx build -t carlo005/dino:latest ./t-rex-runner/ --push
    docker buildx build -t carlo005/tetris:latest ./javascript-tetris/ --push
    docker buildx build -t carlo005/mario:latest ./mario/ --push
    ```

-   Once done, check the images in your [Docker Hub](https://hub.docker.com/) repository.

## **3.	Check out and review the Github Actions Workflow directory and its content. The directory is called .github/workflows/.**
-   
    <div align="left">
    <img src="https://github.com/Carlo-05/Docker-Arcade-Platform-Project/blob/main/documents/pictures/Execute_005.png?raw=true" alt="Execute"style="width: 20%; height: auto;">
    </div>    

-   **master.yml** - manually manages the main branch pipeline and integrates all stages like infra.yml and docker.yml using workflow_dispatch. This feature allows users to run a workflow on demand directly from the GitHub Actions dashboard.
-  **infra.yml** - terraform provisions AWS infrastructure securely using AWS using OIDC authentication.
-  **docker.yml** - execute docker compose to containerized images from Docker Hub repository.
-  **destroy.yml** - use workflow_dispatch to decommission the aws infrastructure built by infra.yml safely.
